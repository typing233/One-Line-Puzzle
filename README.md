# 一笔画 - 创新解谜游戏

一款打破传统直线与网格限制，巧妙融合曲线、单向约束与重复利用机制的创新策略型一笔画解谜游戏。

## 核心特色

### 🎨 打破传统限制
- **曲线连接**：使用贝塞尔曲线连接节点，告别单调直线
- **无网格约束**：节点可自由布局，创造无限可能
- **流畅动画**：精美的视觉效果和通关插画动画

### ⚡ 创新游戏机制
- **单向路径**：带有固定箭头的单向约束，只能沿箭头方向通过
- **重复路径**：带有计数器的路径，可重复使用但有次数限制
- **幻影节点**：特殊节点，访问后自动消失，增加策略深度

### 🎮 游戏体验
- **视觉反馈**：连线发光效果、错误震动动画、实时颜色提示
- **音效系统**：点击音、成功音、错误音，增强沉浸感
- **撤销功能**：支持撤销上一步操作，降低试错挫败感
- **触屏支持**：完美支持鼠标和触屏操作

## 游戏玩法

### 基本操作
1. **开始游戏**：点击起点节点（默认高亮显示）
2. **连接节点**：从当前节点拖拽到相邻节点
3. **完成关卡**：一笔连接所有节点，不重复访问已访问节点

### 特殊机制

#### 🔵 贝塞尔曲线 (蓝色)
- 使用曲线连接节点，视觉更加优美
- 点击和拖拽方式与直线相同

#### 🔴 单向路径 (红色虚线 + 箭头)
- 只能沿箭头方向通过
- 逆向连接会触发错误提示

#### 🟡 重复路径 (橙色虚线)
- 标注了最大使用次数（如 ×2）
- 使用完次数后路径自动断开

#### 👻 幻影节点 (紫色)
- 访问后自动消失
- 增加策略选择的难度

### 控制按钮
- **🔄 重置**：重新开始当前关卡
- **↩️ 撤销**：撤销上一步操作
- **📋 选关**：打开关卡选择界面

## 关卡设计

### 关卡 1 - 初识一笔
- **难度**：★☆☆☆☆
- **目标**：学习基本操作，连接3个节点
- **机制**：仅直线连接

### 关卡 2 - 曲线初探
- **难度**：★☆☆☆☆
- **目标**：体验贝塞尔曲线连接
- **机制**：直线 + 贝塞尔曲线

### 关卡 3 - 单向通道
- **难度**：★★☆☆☆
- **目标**：理解单向路径约束
- **机制**：直线 + 单向路径

### 关卡 4 - 重复路径
- **难度**：★★☆☆☆
- **目标**：掌握重复路径的使用策略
- **机制**：直线 + 重复路径

### 关卡 5 - 幻影挑战
- **难度**：★★★☆☆
- **目标**：应对消失节点的挑战
- **机制**：直线 + 幻影节点

### 关卡 6 - 综合挑战
- **难度**：★★★★★
- **目标**：综合运用所有机制
- **机制**：全部机制融合

## API 接口

### 获取所有关卡
```
GET /api/levels
```
返回所有关卡的详细数据。

### 获取单个关卡
```
GET /api/levels/<level_id>
```
返回指定关卡的详细数据。

### 获取关卡插画
```
GET /api/levels/<level_id>/illustration
```
返回通关后显示的插画数据。

### 验证路径
```
POST /api/verify-path
Content-Type: application/json

{
    "levelId": 1,
    "path": [
        {"from": "n1", "to": "n2"},
        {"from": "n2", "to": "n3"}
    ]
}
```
验证路径是否有效。

## 扩展开发

### 添加新关卡
编辑 `app.py` 中的 `LEVELS` 数组，添加新的关卡配置：

```python
{
    "id": 7,
    "name": "新关卡名称",
    "description": "关卡描述",
    "nodes": [
        {"id": "n1", "x": 150, "y": 200, "type": "normal"},
        // 更多节点...
    ],
    "connections": [
        {"from": "n1", "to": "n2", "type": "straight"},
        // 更多连接...
    ],
    "startNode": "n1",
    "requiredVisits": {"n1": 1, "n2": 1, ...}
}
```

### 添加新机制
在 `game.js` 中的 `OneLinePuzzleGame` 类中扩展：

1. 在 `canTraverseConnection` 方法中添加新的连接类型检查
2. 在 `drawConnections` 方法中添加新的视觉样式
3. 在 `isNodeVisited` 方法中添加新的节点类型逻辑

## 设计理念

### 核心创新点
1. **一笔成画**：保持经典一笔画的核心玩法
2. **曲线美学**：引入贝塞尔曲线，提升视觉体验
3. **策略深度**：通过单向、重复、幻影等机制增加策略性
4. **渐进式难度**：关卡设计从简单到复杂，逐步引入新机制

### 用户体验
- **直观操作**：拖拽连接，符合直觉
- **即时反馈**：错误立即提示，避免迷茫
- **降低挫败**：撤销功能鼓励试错
- **奖励机制**：通关后显示精美插画
